过了这么多天，当前的代码相对于上一个版本还是有一些改变的。首先就是如果你现在不想在本地跑自己的一个千万的小模型的话，你可以接入自己的 API，填入你们的请求地址还有密钥以及模型名称，然后到时候他会自动去请求你，这一个模型。

还有就是现在也添加了两个模型的切换，一个是 1.5B 的模型，还有一个 2B 的模型。如果你们是接受 API 使用的话，这个 1.5B 的模型大概是会用到 6G 左右的一个显存，如果是使用这个 2B 的模型的话，就大概需要用到 9-10G 左右的显存。

虽然说它这个显存变化可能不大，但是经过我自己的测试的话，这个 2B 的模型它这个响应速度是明显快于这个 1.5B 的模型。所以虽然 2B 的模型在流畅度还有连续性上面都会比 1.5B 的模型要好，但是你们也得要考虑一下自己的电脑能不能顶得住。我建议是在你的显卡是在 4070 Super 以上的时候，再去使用这个 2B 的模型。

还有就是对这个资源目录做了一些改变，你们下载下来不是两个压缩包吗？你们到时候下载下来把这个 SHS 给解压，然后你们这里就选中这个 SHS 文件就可以。

## 关键笔记（AI 编译层）

- **项目名称：Whispera** — ASR+LLM+TTS 实时语音对话系统，作者 赚钱养猫猫Cas
- **GitHub**: https://github.com/maomao-2001/Whispera
- **架构**: 语音识别（ASR）→ 大语言模型（LLM）→ 语音合成（TTS），端到端实时对话管道
- **模型选项**:
  - 1.5B 模型：~6GB 显存
  - 2B 模型：~9-10GB 显存，响应更快、流畅度和连续性更好，建议 RTX 4070 Super 以上
- **API 模式**: 支持用户自建 API endpoint，填入请求地址、密钥和模型名即可使用远程 LLM，无需本地跑模型
- **资源目录**: 下载包包含两个压缩包，需要解压 SHS 文件
