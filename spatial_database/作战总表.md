# 空间数据库期末复习包｜作战总表

## 一、这份总表怎么用

这不是一份“从头看到尾”的笔记，而是一份**3 天理论冲刺执行表**。你的老师偏理论，但根据目前拿到的**学长口径题型重点**，复习不能只盯名词解释和简答题，还要兼顾**选择题、填空题、SQL语句题、论述题**。重点分布大致是：选择10分、填空10分、SQL 32分、简答24分、论述24分，另有30分附加题（偏加分/捞分）。所以复习目标不是“每章都看懂全部细节”，而是做到三件事：

1. **知道整门课在讲什么**，脑中有章节地图；
2. **抓住高频理论点**，尤其是定义、流程、比较；
3. **把知识写成答案**，而不是只会看不会答。

这 3 天的最稳妥节奏就是：
- **第 1 天：搭框架** —— 把“数据库基础 → 空间数据库核心”的主线立起来；
- **第 2 天：攻核心** —— 重点吃透模型、设计、存储、索引、查询；
- **第 3 天：练输出** —— 把名词解释、简答、比较题练成考场语言。

如果你时间不够，优先保住每天的 **当日必背** 和 **晚上收口**，这两块最能稳分。

---

## 二、3 天总策略

### 1. 复习顺序为什么这样安排

空间数据库这门课不是零散堆概念，而是有一条很清楚的主线：

**关系数据库基础 → SQL 与约束 → 数据库设计 → 空间数据模型 → 空间数据库建设 → 空间存储与索引 → 空间查询 → 开发与扩展 → NoSQL 与前沿**

如果你一开始就冲进空间索引、空间查询，很容易只记住几个术语，却不知道它们为什么存在。反过来，如果先把数据库基础和课程框架理顺，后面很多知识点都会自动归位。

### 2. 每天都要做的三件事

不管复习哪一章，每天都做这三步：

- **先看框架**：这一章解决什么问题，和前后章节怎么连；
- **再背知识点**：定义、流程、比较点、作用；
- **最后练输出**：至少练几道名词解释或简答题。

这比“只看笔记”稳得多，因为考试不是考你眼熟，而是考你写得出来。

## 三、Day 1：建立框架，拿住基础分

### 1. 今日目标

今天的目标不是死背，而是先把全书框架搭好，解决“脑中没有地图”的问题。你要知道：
- 关系数据库为什么是空间数据库的底座；
- SQL、完整性、安全性、数据库设计为什么是理论高频；
- 后面空间数据库的模型、索引、查询，都是在前面基础上扩展出来的。

### 2. 今日复习章节

建议顺序：
1. 第一讲：空间数据库概述
2. 第二讲：关系数据库原理及关系模型
3. 第三讲、第四讲：SQL 一 / SQL 二
4. 第五讲：数据库完整性、安全性及并发恢复
5. 第六讲：数据库设计

### 3. 时间安排建议

- **上午 8:30–11:30**：第一讲 + 第二讲
- **下午 14:00–17:00**：第三讲 + 第四讲
- **晚上 19:00–21:30**：第五讲 + 第六讲 + 收口

### 4. 当日必背

今天必须记住下面这些：

#### 概念类
- 空间数据库
- 空间数据
- 关系模型
- 关系、元组、属性、域
- 主键、外键、候选码

#### 约束类
- 实体完整性
- 参照完整性
- 用户定义完整性
- 事务 ACID

#### SQL 类
- DDL、DML、DCL
- WHERE 与 HAVING
- 连接查询与子查询

#### 设计类
- 需求分析
- 概念结构设计
- 逻辑结构设计
- 物理结构设计
- 规范化（1NF / 2NF / 3NF）

### 5. 今日训练重点

今天先不大量做题，而是训练“一个概念说完整”的能力。比如你要能用 3 句话说清：
- 什么是关系模型；
- 完整性约束分哪几类；
- 数据库设计为什么要分阶段；
- 空间数据库和关系数据库是什么关系。

建议训练：
- 名词解释 6 题；
- 简答题 3 题；
- 比较题 1 题（如：关系数据库 vs 空间数据库）。

### 6. 晚上收口

今天晚上只做两件事：

1. **画一页总框架图**：
   - 数据库基础
   - SQL 与约束
   - 设计
   - 空间模型
   - 存储与索引
   - 空间查询
   - NoSQL 与前沿

2. **口述课程主线**：
   从“关系数据库怎么组织数据”，讲到“空间数据库怎么表达、存储和查询空间对象”。

### 7. 如果时间不够怎么办

如果今天来不及，至少完成：
- 第二讲 关系模型
- 第五讲 完整性与安全性
- 第六讲 数据库设计

这三块最稳，最容易出理论题。

## 四、Day 2：集中突破空间数据库核心

### 1. 今日目标

今天是全套复习中最关键的一天。重点不是“看得多”，而是要真正搞懂这门课最像专业课的部分：

- 空间对象怎么表达；
- 空间数据库怎么设计与建设；
- 空间数据怎么存；
- 为什么要有空间索引；
- 空间查询和普通查询有什么不同。

### 2. 今日复习章节

建议顺序：
1. 第七讲：空间数据库模型
2. 第八讲：空间数据库设计与建设
3. 第九讲：空间数据库存储与索引
4. 第十讲、第十一讲：空间查询（一）/（二）

### 3. 时间安排建议

- **上午 8:30–11:30**：第七讲 + 第八讲
- **下午 14:00–17:30**：第九讲 + 第十讲
- **晚上 19:00–21:30**：第十一讲 + 收口训练

### 4. 当日必背

#### 模型类
- 点、线、面、体
- 矢量模型
- 栅格模型
- 拓扑关系
- 度量关系
- 方位关系

#### 建设类
- 空间实体识别
- 坐标系统与投影
- 图层组织
- 数据采集
- 数据预处理
- 入库与质量控制

#### 存储索引类
- 空间存储对象
- 最小外接矩形（MBR）
- 网格索引
- 四叉树
- R 树 / R*树

#### 查询类
- 范围查询
- 邻近查询
- 空间连接
- 缓冲分析
- 叠加分析

### 5. 今日训练重点

今天必须开始大练**比较题和简答题**。

建议重点练：
- 矢量模型 vs 栅格模型
- 普通索引 vs 空间索引
- 属性查询 vs 空间查询
- 空间数据库建设流程
- 空间索引为什么重要
- 空间查询执行流程

你答题时要有“统一维度意识”。比如“矢量 vs 栅格”，至少从下面几方面比：
- 表达对象
- 数据结构
- 精度
- 存储量
- 分析特点
- 适用场景

### 6. 晚上收口

今晚的核心任务是把这条主线背顺：

**空间对象先建模 → 再组织与建设 → 再存储 → 再建立索引 → 最后进行空间查询与分析**

你只要能把这条线完整说清，很多简答题自然就能展开。

### 7. 如果时间不够怎么办

如果今天来不及，至少保住三块：
- 第七讲 空间数据库模型
- 第九讲 空间数据库存储与索引
- 第十/十一讲 空间查询

这三块是整门课专业性最强、命题率也很高的部分。

## 五、Day 3：冲刺输出，拿住考场表现

### 1. 今日目标

今天不再追求“学新知识”，而是把前两天的内容变成真正能写上卷面的答案。你要做的是：
- 补齐剩余章节；
- 复盘全书高频理论题；
- 把名词解释、简答、比较题的模板固定下来。

### 2. 今日复习章节

建议顺序：
1. 第十二讲 / 第十三讲：空间数据库系统开发（理论理解）
2. 第十四讲：NoSQL 数据库
3. 第十五讲：空间数据库发展前沿
4. 全书总押题回顾

### 3. 时间安排建议

- **上午 8:30–11:00**：开发 + NoSQL + 前沿
- **下午 14:00–17:30**：名词解释、简答题、比较题专题训练
- **晚上 19:00–21:30**：总押题页复盘 + 临考速背

### 4. 当日必背

#### 开发类
- 空间数据库应用开发流程
- 数据库层 / 服务层 / 应用层
- PostGIS 的作用（理论上理解）
- 空间 SQL 的意义

#### NoSQL 类
- NoSQL 的含义
- 键值 / 文档 / 列族 / 图数据库
- 关系数据库 vs NoSQL
- CAP / BASE（若老师讲过）

#### 前沿类
- 时空数据库
- 云空间数据库
- 大数据 GIS
- 实时位置服务
- 三维空间数据库
- 智能化趋势

### 5. 今日训练重点

今天只做一件最值回票价的事：**把知识练成题型输出**。

建议最低训练量：
- 名词解释：10 题
- 简答题：6 题
- 比较题：4 题

如果时间更紧，优先顺序是：
1. 名词解释
2. 简答题
3. 比较题

### 6. 晚上收口

今晚只看：
- 每讲“小押题”
- 总押题页
- 你还不会的比较题

不要再碰太细的边缘内容。临考前最重要的是：
- 术语准确；
- 结构清楚；
- 能快速分点作答。

### 7. 如果时间不够怎么办

就算只剩半天，也要完成：
- 高频名词解释清单
- 高频简答题清单
- 高频比较题清单
- 考场作答模板

因为最后的分数，常常不是输在“没见过”，而是输在“不会组织答案”。

## 六、考前最后半天怎么压缩

如果你真的只剩最后半天，建议按下面顺序冲：

1. **先看知识地图总框架**（20 分钟）
2. **再背高频名词**（40 分钟）
3. **再过简答题模板和比较题模板**（40 分钟）
4. **最后只看总押题页**（60–90 分钟）

优先级：
- 第一优先：空间数据库特点、矢量/栅格、空间索引、空间查询、数据库设计流程
- 第二优先：完整性与安全性、关系模型、SQL 分类、NoSQL 对比
- 第三优先：开发与前沿

## 七、每天结束时自检什么

每天结束都问自己这 5 个问题：

1. 我能不能用一句话说出这一章在讲什么？
2. 我能不能说出这一章最容易考的 3 个点？
3. 我能不能做 1 道名词解释？
4. 我能不能做 1 道简答题？
5. 我能不能说清它和前后章节的关系？

只要这 5 个问题大多能答上来，你的复习就已经很稳了。

## 八、这份总表最适合什么样的你

这份总表最适合：
- 离考试只剩 3 天；
- 老师偏理论题；
- 不想再无序翻课件；
- 想用最稳的方式把分拿住。

如果你照这个节奏走，再配合《知识地图》和《得分手册》，你的复习就不是“看了很多”，而是“知道考什么、知道怎么答、知道最后怎么冲”。
