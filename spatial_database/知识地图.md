# 空间数据库期末复习包｜知识地图

## 一、先把整门课看成一条主线

空间数据库这门课最怕复习成“每章都背了一点，但彼此断开”。其实它的知识主线很清楚：

**关系数据库基础 → SQL 与约束 → 数据库设计 → 空间数据模型 → 空间数据库建设 → 空间存储与索引 → 空间查询 → 系统开发 → NoSQL 与发展前沿**

所以复习时不要把每一讲看成孤岛，而要不断问自己：
- 这一讲解决什么问题？
- 它是前一讲的延伸，还是后一讲的基础？
- 如果老师出简答题，这一讲最可能问什么？

你要建立的，不是一堆零散词条，而是一张“能串起来”的知识地图。

## 二、第一讲：空间数据库概述

### 这一讲学什么
这一讲解决的是“空间数据库到底是什么”这个总问题。它相当于全课的入口，帮你区分“普通数据库”和“空间数据库”。

### 核心知识点
1. **空间数据库的定义**：用于存储、管理、查询和分析空间数据的数据库系统。
2. **空间数据的组成**：位置、形状、属性，有时还包含时间信息。
3. **空间数据库的核心能力**：支持空间数据类型、空间查询、空间分析、空间索引。
4. **空间数据库与传统数据库的区别**：传统数据库主要管理属性数据，空间数据库还要管理几何对象、空间关系和拓扑关系。
5. **应用场景**：GIS、导航、遥感、智慧城市、位置服务、环境监测等。

### 容易出什么理论题
- 什么是空间数据库？
- 空间数据有哪些特点？
- 空间数据库和传统数据库有何区别与联系？

### 小押题
1. 名词解释：空间数据库、空间数据。
2. 简答：空间数据库相比传统数据库增加了哪些能力？
3. 比较：属性数据与空间数据的区别。

## 三、第二讲：关系数据库原理及关系模型

### 这一讲学什么
这是空间数据库的底层基础。空间数据库很多实现都建立在关系数据库或对象关系数据库之上，所以这一讲一定要掌握关系模型的基本语言。

### 资料状态
- **对应详细笔记**：`详细复习笔记_第1-3-4讲.md`
- **注意**：虽然文件名仍写“第1-3-4讲”，但正文里**已经补入完整第二讲详细内容**，可直接按第二讲部分系统复习。

### 核心知识点
1. **关系模型三部分**：基本结构、基本操作、完整性约束。
2. **关系模型基本概念**：域（Domain）、笛卡尔积（Cartesian Product）、元组（Tuple）、关系（Relation）、关系模式（Schema）。
3. **关系模式 vs 关系**：前者是稳定的结构，后者是某一时刻的数据值。
4. **关系的六大特性**：无重复元组、行序无关、列序无关、同属性同域、属性名不同、属性值原子性（1NF）。
5. **码的概念**：超码、候选码、主码、外码、全码，以及主属性 / 非主属性。
6. **关系完整性**：实体完整性、参照完整性、用户定义完整性。
7. **关系代数基本操作**：选择、投影、连接、并、差、交、笛卡尔积、除、外连接。
8. **并相容性**：并、差、交等集合运算成立的前提条件。

### 容易出什么理论题
- 关系模型由哪三部分组成？
- 关系模式和关系有什么区别？
- 关系有哪些基本特性？
- 主键和外键各自有什么作用？
- 实体完整性与参照完整性的区别是什么？
- 关系代数基本操作有哪些？
- 自然连接和等值连接有什么区别？

### 小押题
1. 名词解释：关系、元组、域、关系模式、主码、外码、并相容性。
2. 简答：关系模型三要素/三部分是什么？
3. 简答：关系的六大特性是什么？
4. 简答：实体完整性与参照完整性的区别是什么？
5. 比较：关系模式 vs 关系，自然连接 vs 等值连接。

## 四、第三讲、第四讲：SQL 一 / SQL 二

### 这一讲学什么
这一部分不是让你死记复杂 SQL，而是让你理解数据库如何定义、操作和控制数据。理论考试更常考 SQL 分类、查询逻辑、连接与子查询的区别。

### 核心知识点
1. **SQL 的三大功能分类**：
   - DDL：定义数据库对象，如 `CREATE`、`ALTER`、`DROP`
   - DML：操作数据，如 `SELECT`、`INSERT`、`UPDATE`、`DELETE`
   - DCL：控制权限，如 `GRANT`、`REVOKE`
2. **基本查询结构**：`SELECT-FROM-WHERE`。
3. **进阶查询内容**：排序、分组、聚合、连接、子查询、视图。
4. **WHERE 与 HAVING 的区别**：前者用于分组前过滤，后者用于分组后过滤。
5. **连接查询与子查询**：都是多表查询的重要方式，但表达方式不同。

### 容易出什么理论题
- SQL 分哪几类，各有什么作用？
- `WHERE` 与 `HAVING` 的区别是什么？
- 什么是视图，它有什么作用？
- 连接查询和子查询各有什么特点？

### 小押题
1. 名词解释：DDL、DML、DCL、视图。
2. 简答：SQL 的主要功能有哪些？
3. 比较：`WHERE` vs `HAVING`，连接查询 vs 子查询。

## 五、第五讲：数据库完整性、安全性及并发恢复

### 这一讲学什么
这一讲是理论考试很爱出的板块，因为它结构固定、概念明确、好分点作答。你要搞清楚：数据库为什么“可信”、为什么“安全”、为什么“能恢复”。

### 核心知识点
1. **完整性约束三类**：
   - 实体完整性：主键不能为空且唯一；
   - 参照完整性：外键必须引用合法主键；
   - 用户定义完整性：由业务规则决定，如 `CHECK` 约束。
2. **安全性机制**：身份认证、授权与回收、视图控制、审计。
3. **并发控制问题**：丢失修改、不可重复读、读脏数据。
4. **事务 ACID**：原子性、一致性、隔离性、持久性。
5. **故障恢复**：日志、检查点、备份与恢复。

### 容易出什么理论题
- 完整性约束有哪些？
- 事务 ACID 分别指什么？
- 并发会带来哪些问题？
- 数据库安全性体现在哪些方面？

### 小押题
1. 名词解释：实体完整性、参照完整性、事务。
2. 简答：事务 ACID 是什么？
3. 比较：完整性与安全性的区别与联系。

## 六、第六讲：数据库设计

### 这一讲学什么
数据库设计回答的是“如何把现实世界合理地变成数据库结构”。它既是关系数据库的重要内容，也是后面空间数据库设计与建设的基础。

### 核心知识点
1. **数据库设计的一般流程**：需求分析 → 概念结构设计 → 逻辑结构设计 → 物理结构设计 → 实施与维护。
2. **ER 模型**：实体、属性、联系，以及 1:1、1:n、m:n 联系。
3. **概念设计与逻辑设计的区别**：前者偏现实抽象，后者偏关系模式实现。
4. **规范化**：减少冗余，避免更新异常，常考 1NF / 2NF / 3NF。
5. **数据库设计的目标**：结构合理、易维护、便于查询、满足需求。

### 容易出什么理论题
- 数据库设计一般分几个阶段？
- ER 图中有哪些基本元素？
- 规范化的目的是什么？

### 小押题
1. 名词解释：ER 模型、规范化、逻辑结构设计。
2. 简答：数据库设计的主要步骤是什么？
3. 比较：概念结构设计 vs 逻辑结构设计。

## 七、第七讲：空间数据库模型

### 这一讲学什么
这一讲标志着课程正式进入空间数据库核心。它要回答：空间对象如何表达？不同模型分别适合什么数据和分析任务？

### 核心知识点
1. **空间对象类型**：点、线、面、体。
2. **矢量模型**：适合离散对象，如道路、河流、建筑物、行政边界。
3. **栅格模型**：适合连续场，如高程、温度、遥感影像、污染扩散。
4. **空间关系三类**：
   - 拓扑关系：相交、包含、相邻、相离；
   - 度量关系：距离、长度、面积；
   - 方位关系：东南西北、左右上下。
5. **矢量与栅格的优缺点**：这是高频比较题。

### 容易出什么理论题
- 什么是矢量模型，什么是栅格模型？
- 拓扑关系为什么重要？
- 矢量模型与栅格模型各适合什么场景？

### 小押题
1. 名词解释：矢量模型、栅格模型、拓扑关系。
2. 比较：矢量模型 vs 栅格模型。
3. 简答：空间关系主要包括哪些类型？

## 八、第八讲：空间数据库设计与建设

### 这一讲学什么
这一讲是在传统数据库设计基础上，进一步考虑空间数据的组织、采集、预处理和入库。它最适合出流程题和原则题。

4. **图层组织与属性结构设计**：让空间对象更易管理与查询。
5. **空间数据质量**：精度、一致性、现势性、完整性。

## 九、第九讲：空间数据库存储与索引

### 这一讲学什么
这是全课最核心、最容易出大题的部分之一。因为空间数据复杂、维度高、关系多，传统索引不够用，所以必须引入专门的空间索引。

### 核心知识点
1. **空间数据存储内容**：属性数据、几何数据、边界框、拓扑关系等。
2. **空间存储的特点**：数据量大、结构复杂、查询代价高。
3. **空间索引的作用**：缩小检索范围、提高查询效率。
4. **常见空间索引类型**：
   - 网格索引
   - 四叉树
   - R 树 / R*树
5. **MBR（最小外接矩形）**：空间索引里常考核心概念。
6. **空间查询为什么比普通查询复杂**：因为要处理几何对象和空间关系。

### 容易出什么理论题
- 为什么需要空间索引？
- R 树的基本思想是什么？
- MBR 在空间索引中的作用是什么？
- 普通索引和空间索引有何区别？

### 小押题
1. 名词解释：空间索引、MBR、R 树。
2. 简答：空间索引为什么重要？
3. 比较：普通索引 vs 空间索引。

## 十、第十讲、第十一讲：空间查询（一）/（二）

### 这一讲学什么
空间查询是前面模型、设计、存储、索引最终落地的地方。普通数据库查的是属性条件，空间数据库还要查位置关系、距离关系、邻近关系。

### 核心知识点
1. **空间查询的主要类型**：
   - 范围查询
   - 空间关系查询
   - 距离查询
   - 最近邻查询
   - 缓冲分析
   - 叠加分析
   - 空间连接
2. **空间查询的一般执行流程**：
   - 先利用空间索引粗筛；
   - 再对候选对象做精确判断；
   - 最终返回结果。
3. **常见空间函数思想**：
   - `ST_Intersects`：是否相交
   - `ST_Contains`：是否包含
   - `ST_Within`：是否在内部
   - `ST_Distance`：距离
   - `ST_Buffer`：缓冲区
4. **属性查询 vs 空间查询**：这是很经典的比较题。

### 容易出什么理论题
- 空间查询有哪些类型？
- 最近邻查询的基本思路是什么？
- 缓冲分析和叠加分析分别解决什么问题？
- 空间查询为什么要借助空间索引？

### 小押题
1. 名词解释：空间查询、缓冲分析、空间连接。
2. 简答：空间查询的一般处理流程。

## 十二、第十四讲：NoSQL 数据库

### 这一讲学什么
这一讲通常作为扩展内容，但很适合出概念题和比较题。重点是理解它为什么出现，以及它和关系数据库是什么关系。

### 核心知识点
1. **NoSQL 的含义**：Not Only SQL。
2. **NoSQL 的出现背景**：海量数据、高并发、弱结构化数据、分布式场景。
3. **常见类型**：键值数据库、文档数据库、列族数据库、图数据库。
4. **NoSQL 与关系数据库的差异**：
   - 关系数据库：结构严格、一致性强、适合事务型业务；
   - NoSQL：模式灵活、扩展性强、适合大规模与非结构化场景。
5. **CAP / BASE（若老师讲过）**：属于理论补充分。

### 容易出什么理论题
- 什么是 NoSQL？
- NoSQL 为什么会出现？
- NoSQL 与关系数据库有什么区别？

### 小押题
1. 名词解释：NoSQL。
2. 简答：NoSQL 的主要特点是什么？
3. 比较：关系数据库 vs NoSQL。

## 十三、第十五讲：空间数据库发展前沿

### 这一讲学什么

## 十四、跨章节总串联

### 1. 从关系数据库到空间数据库
你要会说：空间数据库不是凭空出现的，而是在关系数据库基础上，为了处理空间对象和空间关系而扩展出来的。

### 2. 从设计到查询的完整链条
你要会说：先有数据库设计，再有空间数据库建设；对象入库后需要存储结构和索引，最后才能高效地完成空间查询与分析。

### 3. 从课程基础到前沿趋势
你要会说：前面的关系模型、SQL、设计和索引，构成了今天空间数据库系统的基础；后面的 NoSQL、云、时空和智能化，是它继续发展的方向。

## 十五、当前已知考试题型重点（学长口径版）

> **重要说明**：这份信息应理解为“老师强调的重点分布”，**不是说考试只考这些**。也就是说，后面列的是复习时的火力重点，不是排他性考纲。

### 题型与重点分布
1. **选择题**：10题，共10分 —— 重点看**基本概念**。
2. **填空题**：10题，共10分 —— 重点覆盖**关系数据库 + 空间数据库**基础概念。
3. **SQL语句题**：8题，共32分 —— 重点是**学懂，不是背原题**。
   - 传统关系数据库几套题要会变式；
   - 空间数据库查询与空间函数也要会基本写法与理解。
4. **简答题**：4题，共24分 —— 重点分布为**关系数据库 2题 + 空间数据库 2题**。
5. **论述题**：2题，共24分 —— 重点看你对知识的**理解、思路与展开能力**。
6. **附加题**：1题，共30分 —— 可看作加分/捞分题，不作为当前主攻核心。

### 这份题型重点对复习策略的影响
- **第二讲到第四讲**：不只是要会概念，还要顶住**填空题 + 传统关系数据库SQL变式题**。
- **第七讲到第十一讲**：不仅会名词和简答，还要能应对**空间查询、空间函数、空间数据库题**。
- **论述题**：不能只背定义，必须会把“为什么、怎么做、适用场景、优缺点、联系区别”展开成完整答案。
- **附加题**：当前不作为主线投入重点，优先保证基础分和24分论述题。

4. 数据库设计一般分哪些阶段？

### 比较题高频
1. 关系数据库 vs 空间数据库
2. 矢量模型 vs 栅格模型
3. 完整性 vs 安全性
4. 普通索引 vs 空间索引
5. 属性查询 vs 空间查询
6. 关系数据库 vs NoSQL
7. 概念结构设计 vs 逻辑结构设计

### 论述题高频（12分/题，需展开论述+举例）
1. **试述矢量模型与栅格模型的区别、优缺点及各自适用场景。**（第7讲核心）
   - 答题框架：数据结构→精度→存储量→分析特点→适用场景→互补关系
2. **什么是空间索引？为什么空间数据库需要空间索引？请阐述R树的基本原理。**（第9讲核心）
   - 答题框架：空间查询的困难→普通索引为什么不行→空间索引如何缩小范围→R树的MBR、层次结构、插入/搜索原理
3. **试述数据库设计的一般流程，并说明空间数据库设计相比传统数据库设计需额外考虑哪些问题。**（第6讲+第8讲）
   - 答题框架：五阶段流程→每阶段解释→空间数据库特殊考虑（几何对象、坐标系统、图层组织、空间关系）
4. **试述空间数据库应用开发的一般流程，并说明各层次（数据库层/业务逻辑层/应用层）的职责。**（系统开发1-3）
   - 答题框架：需求分析→设计→建库→开发→部署→各层职责→举例（如GIS系统）
5. **空间查询有哪些主要类型？请阐述空间查询的一般执行流程及其与属性查询的区别。**（第10-11讲）
   - 答题框架：分类列举（范围/邻近/空间连接/缓冲/叠加）→两步执行流程（粗筛→精判）→与属性查询对比
6. **试述NoSQL与关系数据库的主要区别，并结合空间大数据场景说明NoSQL的作用与局限。**（第14讲）
   - 答题框架：NoSQL含义→出现背景→四类+对比→空间大数据场景→优势+局限→混合持久化结论

## 十六、怎么用这份知识地图

最稳的用法是：
1. 先通读每讲“这一讲学什么”；
2. 再背“核心知识点”；
3. 最后立即做“容易出什么理论题”和“小押题”。

如果你做到“每一讲都能讲清：它讲什么、为什么重要、最可能怎么考”，那你这门课的理论框架就已经搭稳了。
