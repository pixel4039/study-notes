# 阅读指南(Obsidian 阅读通道 A)

> 说明:本页由 Sonnet 按 Claude 拟定的合同(`阅读通道 A` 交付合同)代笔执行落地,frontmatter 按仓库 SCHEMA 要求署 `author: claude`;文字与配置内容已由 Claude 验收。

本仓库(`study-notes`)选定的阅读方式是「通道 A」:电脑本地 Obsidian + git clone + Obsidian Git 插件,让 GitHub 仓库本身就是 Obsidian vault,笔记与留言通过 git 提交/推送同步,不额外引入云盘或第三方笔记服务。

## PC 端(优先)

1. 安装 [Obsidian](https://obsidian.md)。
2. 用 `git clone git@github.com:pixel4039/study-notes.git`(或 GitHub Desktop 图形化 clone)把仓库拉到本地一个目录。
3. 打开 Obsidian → "Open folder as vault" → 选择上一步 clone 出来的目录本身作为 vault(不要再套一层子目录)。
4. 打开 Obsidian 设置 → Appearance → CSS snippets,确认 `reading` 已勾选启用(仓库已在 `.obsidian/appearance.json` 里预置 `enabledCssSnippets: ["reading"]`,正常情况下打开 vault 就是启用状态;如果显示未启用,手动勾选一次即可)。
5. 安装社区插件 **Obsidian Git**(仓库本身不预置任何社区插件配置文件,插件安装与个人偏好设置都在你自己电脑上完成,不会写回仓库配置)。建议配置:
   - Auto pull:vault 启动时自动 pull;
   - Auto push:每 10 分钟自动 push,或改成手动点击 push(两种都可以,看你希望多实时)。
6. 日常阅读入口 = 每条视频对应的**连续阅读**稿(`sources/bilibili/<MID>/transcripts/<year>/...__连续阅读.md`),不是逐句加时间戳的 canonical 字幕稿(那份是不可变原始层,用来核对用词,不作为默认阅读版面)。
7. 留言写法:想留言时,直接打开该视频对应的 annotation 页(`annotations/bilibili/<MID>/<BVID>.md`),把想法追加到「留言栏(用户原话)」段落末尾即可,不新建文件、不改已有留言。
8. 写完存盘后,让 Obsidian Git 插件把改动 push 回 GitHub(自动或手动都行),Codex/Claude 下次处理入库时会读到。

## 移动端(iPad / 手机,体验有限,PC 优先)

- iPad/手机可以装 Obsidian,再打开同一个 GitHub 仓库作为 vault。
- iOS 上 git 同步推荐两条路:
  - **Working Copy**(专门的 iOS git 客户端,配合 Obsidian 用文件 Provider 打开同一份 clone);
  - 或 **Obsidian Git 插件的移动端版本**(功能和易用性都弱于桌面版,冲突处理更麻烦)。
- 这条路线目前只作为补充,遇到冲突或体验问题优先回到 PC 端处理,不必强求移动端功能对等。

## 分支说明

远端默认分支是 **master**,不是 `main`;clone、pull、push 都对着 `master` 操作即可,不需要额外切分支。
